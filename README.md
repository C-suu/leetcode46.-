### （1）题目白话文解释与生僻概念说明

**题目大意：**
给定一组互不相同的数字，要求把这些数字所有的排列方式都列举出来。比如给出数字 1、2、3，那么它们组合出来的所有可能情况有：123、132、213、231、312、321。

**生僻/核心概念解释：**

  * **全排列 (Permutation)：** 这是一个组合数学概念，指的是从给定的所有元素中，按照一定的顺序取出所有元素进行排列的过程。
  * **深度优先搜索 (DFS - Depth First Search)：** 一种遍历策略。在排列组合中，它的表现是“一条道走到黑”。先固定第一个数字，再固定第二个数字，一直往深处走，直到凑齐一个完整的排列方案，然后再退回来换下一条路。
  * **回溯 (Backtracking)：** 回溯是深度优先搜索中的核心操作。可以将其理解为“走不通就退回上一个十字路口”。当一条路径走到底，得到了一个结果后，程序需要“撤销”上一步的操作，把状态恢复到之前的样子，才能去尝试其他的选择分支。

### （2）代码解题思路解析

从代码的角度反推，这段逻辑采用了一种\*\*“原地交换 (Swap-based)”\*\*的回溯方法，非常巧妙地节省了额外的空间。

**具体思路如下：**

1.  **坑位固定法：** 假设有 `n` 个数字，就可以看作有 `n` 个空位。函数 `dfs(x)` 的含义是：现在准备往第 `x` 个空位上放数字。
2.  **枚举选择：** 对于第 `x` 个空位，它后面（包括它自己）的所有数字都有资格坐这个位置。因此用一个 `for` 循环，枚举索引从 `x` 到末尾的所有数字。
3.  **交换与深入：** 把选中的数字（索引为 `i`）与当前坑位（索引为 `x`）的数字进行交换。交换后，第 `x` 位的数字就“固定”下来了。接着，调用 `dfs(x + 1)`，去安排下一个坑位。
4.  **撤销交换（回溯的核心）：** 当 `dfs(x + 1)` 执行完毕返回时，说明基于刚才那种选择的所有后续排列都已经找完了。此时，**必须把刚才交换的两个数字再换回来**，让数组恢复原状。只有这样，`for` 循环进入下一次迭代时，才能在一个干净、原始的基础上做出新的选择。
5.  **终止条件：** 当 `x` 来到数组的最后一个位置时，说明前面的坑位都填满了，只剩这一个数字，也就没有交换的必要了。此时，整个数组刚好构成了一个新的完整排列，将其记录下来即可。

### （3）代码框与逐行注释

```python
class Solution:
    def permute(self, nums: List[int]) -> List[List[int]]:
        def dfs(x):                                 # 定义内部深度优先搜索函数，x 表示当前正在确定数组第 x 位的值
            if x == len(nums) - 1:                  # 当 x 到达数组最后一个元素的索引时，说明一个完整的排列已经生成
                res.append(list(nums))              # 将当前数组的状态复制一份成新列表，并添加到结果集 res 中
                return                              # 结束当前这一层递归，返回上一层
            for i in range(x, len(nums)):           # 遍历从第 x 位到最后一位的所有元素，尝试将它们放到第 x 位
                nums[i], nums[x] = nums[x], nums[i] # 做出选择：将第 i 位元素与第 x 位元素交换，固定第 x 位
                dfs(x + 1)                          # 进入下一层递归：继续去确定第 x + 1 位的值
                nums[i], nums[x] = nums[x], nums[i] # 撤销选择：递归返回后，将刚刚交换的元素换回来，恢复数组原状
        res = []                                    # 初始化一个空列表，用于存放最终所有的排列组合结果
        dfs(0)                                      # 从第 0 位开始触发深度优先搜索
        return res                                  # 返回包含所有排列组合的结果集
```

### （4）每一行代码的详细中文解释

  * `class Solution:`
    定义用于封装算法的类。
  * `def permute(self, nums: List[int]) -> List[List[int]]:`
    定义主函数 `permute`，接收一个整数列表 `nums`，返回一个二维列表（里面包含所有的排列组合）。
  * `def dfs(x):`
    在主函数内部定义一个辅助的递归函数 `dfs`。参数 `x` 代表当前递归正在处理数组中的第几个位置（索引）。
  * `if x == len(nums) - 1:`
    判断当前的索引 `x` 是否已经到达了数组的最后一位（即总长度减 1）。如果是，说明前面的所有位置都已经通过交换固定好了，一个全新的排列诞生了。
  * `res.append(list(nums))`
    将当前形成的排列添加到结果列表 `res` 中。注意这里使用了 `list(nums)`。因为 `nums` 是一个可变对象，如果在后续的代码中它被修改了，直接 `append(nums)` 会导致之前添加的结果也被连带修改。使用 `list()` 可以创建当前数组状态的一个完整副本，安全地存入结果集中。
  * `return`
    完成了一个排列的记录后，结束当前的函数执行，使程序流程退回到调用 `dfs` 的上一层。
  * `for i in range(x, len(nums)):`
    开启一个循环，变量 `i` 从当前的坑位索引 `x` 一直遍历到数组的末尾。这意味着，位于 `x` 及其之后的所有元素，都有机会被放到第 `x` 位上。
  * `nums[i], nums[x] = nums[x], nums[i]`
    Python 特有的平行赋值语法，用于交换两个元素的位置。把索引为 `i` 的元素调换到当前的坑位 `x` 上，代表“做出了一个选择”。
  * `dfs(x + 1)`
    当前第 `x` 位固定好之后，调用自身，把处理目标推进到下一个位置（`x + 1`），继续重复上述固定元素的逻辑。
  * `nums[i], nums[x] = nums[x], nums[i]`
    核心的回溯动作。当 `dfs(x + 1)` 执行完毕并返回后，说明刚才做出的选择已经穷尽了后续的所有可能。此时再次执行交换动作，将元素位置还原。这样可以确保下一次循环挑选新元素时，面对的是一个未被污染的原始数组状态。
  * `res = []`
    在主函数中初始化一个空列表 `res`，用来充当收集所有最终排列方案的容器。
  * `dfs(0)`
    正式启动回溯算法，传入初始参数 `0`，表示从数组的第 0 个位置开始安排元素。
  * `return res`
    当所有可能的排列都被遍历、记录，且 `dfs(0)` 完全执行结束后，返回装满排列方案的 `res` 列表。
